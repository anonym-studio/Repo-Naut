# Zenn記事 構成案

**タイトル（案）**:
> 思いつきでアプリ開発を始めるせいでローカルリポジトリがとっちらかって大変なので、リポジトリを一覧で確認できて各ディレクトリ起点でターミナルやエディタを開いたり簡単なgit操作ができたりプロジェクトをアーカイブしたりできるデスクトップアプリをつくった

**トピックタグ案**: `#tauri` `#rust` `#react` `#個人開発` `#デスクトップアプリ`

## 構成案

---

### 1. はじめに（問題提起）

**伝えたいこと**: 「あるある」として共感を取る。記事タイトルを補足する形で課題を具体的に描写する。

**内容案**:

- 「思いつくたびにリポジトリを作る」癖がある → いつの間にかローカルにプロジェクトが何十個も溜まっている
- Finder でプロジェクトを探し回る、どのディレクトリにあるかわからない
- `cd ~/projects && ls` して `cd` して `cursor .` するループが面倒
- 古いプロジェクトが邪魔で整理したいが、削除するのは怖いのでずっと放置
- タスクは別ツール（Notion / Linear など）に書くが、リポジトリとの紐付けがない

→ 「ないなら作ろう」という流れへ

---

### 2. つくったもの

**伝えたいこと**: 何ができるアプリかを一言＋スクリーンショットで伝える。

**内容案**:

- アプリ名: **Repo-Naut**（ランディングページ: https://repo-naut.pages.dev/）
- 一言: 「ローカルの Git リポジトリをカード形式で一元管理するデスクトップアプリ」
- GitHub Releases から macOS 向け `.dmg` を配布中
- スクリーンショット（assets/screen-*.png を使用）を載せる
  - Repos 一覧（カードビュー）
  - カンバンボード
  - ダッシュボード（ヒートマップ）
  - アーカイブ一覧

---

### 3. 主な機能の紹介

**伝えたいこと**: 「これが欲しかった」と思ってもらえる機能を具体的に紹介する。

#### 3-1. ワークスペーススキャン（一覧表示）

- ルートディレクトリを登録するだけで配下の Git リポジトリを自動検出・カード表示
- 最新コミットメッセージ・ブランチ名・ahead/behind・未コミット数をリアルタイム表示
- `.git/HEAD` の変更を監視して自動再スキャン（コミットしたら即反映）
- 複数 Workspace 登録可能（仕事 / 個人 / OSS など切り替え）

#### 3-2. エディタ・ターミナルを即起動

- VS Code / Cursor / Zed などをプリセット登録、カードからワンクリック起動
- ターミナルも同様（iTerm2 / Terminal.app / Ghostty など）
- 「エディタを開いてプロジェクトを始める」までの操作を最短に

#### 3-3. Git 操作（pull / fetch / ブランチ切替）

- カードから `git pull` / `git fetch` を実行、結果をモーダルで確認
- ブランチ一覧・切り替えもアプリ内で完結

#### 3-4. アーカイブ機能

- 古いプロジェクトを `_archive/` ディレクトリへ移動して整理
- スナップショットメタ（最終コミット・日時・元のパス）を保存
- 「完全削除はできないけど邪魔なもの」を整理できる

#### 3-5. リポジトリ横断カンバン

- Todo / In Progress / Review / Done の 4 カラム
- タスクをリポジトリに紐付けられる（「どのプロジェクトのタスクか」が一目瞭然）
- @dnd-kit でドラッグ＆ドロップ対応

#### 3-6. その他（コンパクトに）

- GitHub PAT 連携（PR 数・Issue 数バッジ表示）
- コマンドパレット（`Cmd+K`）+ Vim 風キーボードナビ
- コミット数ヒートマップ（直近 30 日）
- タグ・メモ（Markdown）
- データのエクスポート / インポート

---

### 4. 技術スタックと選定理由

**伝えたいこと**: Tauri を選んだ理由・Rust を使った部分の説明を、エンジニア向けに簡潔に。

#### Tauri v2 を選んだ理由

- Electron より**メモリ消費が圧倒的に少ない**（本アプリは常駐ツールなのでここは重要）
- フロントエンドは慣れた React + TypeScript で書ける
- Rust でネイティブ機能（Git 操作・ファイル監視・キーチェーン）が書ける

#### 技術スタック早見表

| レイヤー | 採用技術 |
|---|---|
| フレームワーク | Tauri v2 |
| フロントエンド | React 18 + TypeScript + Vite + Tailwind CSS v4 |
| 状態管理 | TanStack Query（サーバー状態）+ Zustand（UI状態） |
| DnD | @dnd-kit |
| バックエンド（Rust） | git2 / notify / rayon / keyring / reqwest |
| データ永続化 | JSON ファイル 3 本（SQLite 不使用） |

### 5. 実装で面白かった / 工夫した点

**伝えたいこと**: 読者が「なるほど」「自分もやってみたい」と思える技術的な話を 3〜4 点。

#### 5-1. Rust の rayon でリポジトリスキャンを並列化

- ワークスペース配下を再帰的に `.git` ディレクトリで検索
- `rayon` の並列イテレータで処理 → リポジトリが 100 件超えても体感的に速い
- コードスニペット（`scan_workspace` のコア部分）

#### 5-2. `notify` クレートでファイル変化を検知 → 自動再スキャン

- `.git/HEAD` と `.git/refs/heads/` だけを監視（`node_modules` や `target` のノイズは無視）
- 800ms debounce で TanStack Query の `invalidateQueries` を叩く
- コミットしたら画面が自動更新される体験が気持ちいい

#### 5-3. GitHub PAT を OS キーチェーンに保存（`keyring` クレート）

- JSON に平文で書かない → `keyring` クレートで macOS Keychain / Windows Credential Store へ保存
- バックアップエクスポートにも含めない設計

#### 5-4. カンバン DnD の order 管理（f64 中間挿入）

- カラム間・同カラム内どちらもドラッグ&ドロップで並び替え可能
- order を整数（0, 1, 2...）にすると移動のたびに全件更新が必要 → `f64` で中間挿入
  - 例: 1.0 と 2.0 の間に挿入 → 1.5。さらに → 1.25
- 精度が尽きたら再採番（実用上はほとんど発生しない）

#### 5-5. pnpm workspace でデスクトップアプリと LP を同居

- ルート: Tauri デスクトップアプリ
- `landing/`: Cloudflare Pages 向けの静的 LP
- `pnpm install` 一発で両方のパッケージが揃う
- ビルド出力は互いに干渉しない（`dist/` vs `landing/dist/`）

### 6. 苦労した点

**伝えたいこと**: 詰まったポイントを正直に書くことで読者の共感・信頼を得る。

- **Tauri v2 のドキュメントが薄い**: v1 からの移行で API が変わっており、サンプルが少ない
- **`generate_handler![]` 登録漏れ**: 新しい Tauri コマンドを追加するたびに `lib.rs` への登録を忘れて実行時エラー → チェックリスト化で対処
- **macOS Gatekeeper 問題**: Apple Developer Program 未加入のため未署名配布。初回起動手順の案内を丁寧にする必要がある
- **Tailwind CSS v4 の情報が少ない**: 設定方法が v3 と大きく変わっていてハマる箇所があった
