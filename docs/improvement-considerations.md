# 改善検討メモ

未実装の改善案・既知の課題・調査結果を蓄積するライブドキュメント。  
仕様書（`spec/`）やタスク履歴（`docs/tasks.md`）とは別に、**「やるかどうか」「いつやるか」未確定の検討事項**を置く。

---

## 運用ルール

| 項目 | 内容 |
|------|------|
| **いつ追記するか** | バグ調査・UX 相談・パフォーマンス議論・プラットフォーム制約の調査で、**即実装しないが将来検討したい**内容が出たとき |
| **誰が追記するか** | 開発者・AI エージェント（作業の最後に 1 件でもよいので追記を推奨） |
| **形式** | 下記「トピックの書き方」に従い、**新しいトピックは末尾に追加**（古い議論は削除せず、ステータス更新で対応） |
| **実装したら** | ステータスを `implemented` にし、PR / コミットへのリンクを `Notes` に書く。詳細仕様は spec 側へ移す |

### トピックの書き方

各トピックは次の見出しレベルで統一する。

```markdown
## [STATUS] タイトル（短く）

- **発生日 / 最終更新**: YYYY-MM-DD
- **優先度**: high | medium | low | unknown
- **影響範囲**: 例）macOS / workspace スキャン / 全 OS
- **関連コード**: パスやコマンド名

### 背景・現象
（ユーザーが困っていること、再現条件）

### 調査メモ
（原因仮説、OS・ライブラリの挙動、関連リンク）

### 対応案
| 案 | 概要 | コスト | 備考 |
|----|------|--------|------|

### ユーザー向け回避策（コード変更なし）
（設定でできること、運用での回避）

### Notes
（実装時のメモ、追記ログ）
```

**STATUS** の例: `open`（未着手） / `investigating` / `planned` / `wontfix` / `implemented`

## [open] macOS — リムーバブルボリューム（USB SSD）上の workspace で許可ダイアログが繰り返し出る

- **発生日 / 最終更新**: 2026-05-17
- **優先度**: medium
- **影響範囲**: macOS（ワークスペースが外付け・リムーバブルボリューム上にある場合）
- **関連コード**:
  - `src-tauri/src/commands/workspace.rs` — `scan_workspace`（`WalkDir` 再帰 + `rayon` 並列）
  - `src-tauri/src/watcher.rs` — `notify` 再帰監視 → `workspace_changed` イベント
  - `src/hooks/useRepos.ts` — `scan_workspace` の react-query
  - `src/hooks/useWorkspaceWatcher.ts` — イベント受信後の `invalidateQueries(['repos'])`
  - `src/components/layout/Onboarding.tsx` / `src/pages/Settings.tsx` — `tauri-plugin-dialog` のフォルダ選択

### 背景・現象

ワークスペースのルートが USB SSD などのリムーバブルボリューム上にあると、起動時や操作中に

> 「Repo-Naut がリムーバブルボリューム上のファイルにアクセスしようとしています」

に類する macOS の TCC 確認が **何度も** 表示される。

### 調査メモ

1. **フルディスクアクセス（FDA）との違い**
   - 本現象は **プライバシーとセキュリティ → ファイルとフォルダ → リムーバブルボリューム** のカテゴリに近い。
   - FDA（`SystemPolicyAllFiles`）はより広い権限で、リムーバブル系を包含する場合があるが、**リポジトリ管理アプリに FDA を付けるのは過剰**。
   - FDA を付けるとプロンプトは減ることがあるが、**推奨される第一選択ではない**。

2. **macOS 側の既知挙動**
   - 許可後も同ダイアログが連続する報告がある（TCC / OS 側の不調の可能性）。
   - `cargo tauri dev` ではバイナリパスが変わると **別アプリ扱い** になり、許可がリセットされやすい。署名済み `.app` でも再現するか切り分けが必要。

3. **アプリ側の要因（プロンプトを増やしやすい構造）**
   - ワークスペースパスは `settings.json` に **文字列のみ** 保存。セキュリティスコープ付きブックマーク等は未実装。
   - `scan_workspace`: `WalkDir`（`max_depth(8)`）+ リポジトリごと `rayon` 並列 + `collect_languages`（リポジトリあたり最大 800 ファイル、`max_depth(3)`）。
   - 起動時に `notify` でワークスペース全体を **再帰監視**（`macos_fsevent`）。
   - `workspace_changed` → debounce 後に `['repos']` を invalidate → **再スキャン**。
   - Dashboard の `get_commit_activity` がワークスペース直下を再度読み込み。
   - オンボーディングではフォルダピッカーがあるが、**パス手入力のみ** でも登録可能（ダイアログ経由の許可と一致しない可能性）。

### 対応案

| 案 | 概要 | コスト | 備考 |
|----|------|--------|------|
| A. ドキュメント・UI のみ | 外付け利用時は「フォルダを選択」必須、システム設定でリムーバブルボリュームをオン、とヘルプに記載 | 小 | 即効性は限定的 |
| B. security-scoped bookmark | フォルダピッカー選択時に macOS のブックマークを保存し、アクセス前に `startAccessingSecurityScopedResource` | 中〜大 | サンドボックス有無・Tauri の API 調査が必要 |
| C. スキャン負荷の低減 | `rayon` の並列度制限、初回フル／以降差分、外付け時は `staleTime` 延長や手動リフレッシュ | 中 | プロンプト回数削減向き。根本の TCC 免除ではない |
| D. 監視の絞り込み | `watcher.rs` の `is_path_relevant` 強化（`.git/objects` 等）、debounce 延長 | 小〜中 | 他ツールによる FS ノイズ対策 |
| E. 手入力パス禁止 | workspace 追加はピッカー経由のみに制限 | 小 | UX はやや厳しくなる |

### ユーザー向け回避策（コード変更なし）

1. **システム設定 → プライバシーとセキュリティ → ファイルとフォルダ → リムーバブルボリューム** で Repo-Naut を許可（FDA は最終手段）。
2. ワークスペース追加は **必ず「フォルダを選択」** から行う（パス手入力のみは避ける）。
3. USB 上で Spotlight・バックアップ・他 IDE が大量に触ると `workspace_changed` が増える — 可能なら当該ボリュームをインデックス対象外にする。
4. 開発中は `pnpm tauri dev` と **署名済みリリース `.app`** の両方で挙動を比較する。
